reverse-to

Synopsis

reverse-to ("bookmark"|"comment"|step|cycle)

Description

Run the simulation in reverse. The simulation will stop at any breakpoint or until the specified point in time if no breakpoint occurred.

One of the following arguments is required. An existing bookmark. A comment made using the add-session-comment command. An absolute time defined by step or cycle.

At least one time bookmark has to be set before any reverse operations are possible.
