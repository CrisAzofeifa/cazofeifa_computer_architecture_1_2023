os_awareness

Description

OS Awareness framework for controlling the trackers

Interfaces Implemented

conf_object, log_object, temporal_state, osa_node_tree_query, osa_control_v2, osa_control, osa_machine_notification, osa_machine_query, osa_tracker_state_notification, osa_tracker_state_query, osa_tracker_state_admin, osa_node_tree_admin, osa_node_tree_notification, osa_component, osa_parameters, osa_node_path

Commands for this class

  • active-node – list the active nodes
  • break-enter – deprecated break the simulation when a processor becomes active on the node
  • break-exit – deprecated break the simulation when a processor becomes inactive on the node
  • delete-tracker – delete tracker component
  • disable-tracker – stop using software tracking
  • enable-tracker – enable software tracking
  • find – find a node
  • info – print information about the object
  • insert-tracker – insert a new child tracker
  • list – list all processes/tasks
  • node-info – print all properties of a software node
  • node-tree – list software node tree
  • status – print status of the object
  • unbreak – deprecated cancel a software breakpoint
  • wait-for-activated – deprecated wait for a processor to become activated on the node
  • wait-for-deactivated – deprecated wait for a processor to become deactivated on the node

Commands for interface osa_component

  • bp-break – break on OS Awareness node changes
  • bp-run-until – run until OS Awareness node changes
  • bp-trace – trace OS Awareness node changes
  • bp-wait-for – wait for OS Awareness node changes

Commands for interface osa_parameters

  • load-parameters – load tracker parameter settings
  • save-parameters – save parameters
  • supports-parameters – check if supported
