Simics Release Notes

1 Minor Changes Since Simics 6.0.0

The following is a list of changes since the initial Simics 6 release.

  • 6.0.166 (build 6218)

    • Classic GUI
      • Fixed a problem, introduced in Simics-Base 6.0.163, where sessions and checkpoints could not be loaded (bug #14019318121).
    • Core
      • Fixed a problem with Python wrappings in interfaces, where the wrong wrapping was used when a type was Python wrapped in several interfaces.
      • Simics now sets the environment variables PYTHONNOUSERSITE and PYTHONVERBOSE to 1.
      • Fixed a problem introduced in Simics-Base 6.0.163 where CLI variables were local to the script, when it was run from the command line (bug #15013379466).
      • The documentation and error messages for wait-for-global-time have been improved.
    • Debugger
      • Minor updates to Simics object handling.
    • Device Modeling Language (DML) 1.2
      • If a bank declared with use_io_memory=false in a DML 1.4 file is imported from a DML 1.2 device, using the `dml12-compatibility.dml` library, then a call to the transaction_access method will now trigger a proper bank access. Also, if a bank uses the dml12_compat_io_memory_access template, then overrides of the transaction_access method will be honored. This simplifies future transition to the Simics 7 API.
    • Device Modeling Language (DML) 1.4
      • Fixed a bug where a top-level object named the same as the device object could result in invalid generated C when both objects are targets of each .. in .. expressions (bug #SIMICS-21044).
    • Target parameters
      • The params.dump command is now deprecated. Use params.list with the -yml flag instead.
      • The documentation key in parameter definitions is now deprecated. Use the description key instead.
      • Added -all argument to the params.save command, for saving not only non-default values.

  • 6.0.165 (build 6217)

    • Project Makefiles
      • Fixed a bug that caused build errors when compiling DML modules with D=1.

  • 6.0.164 (build 6216)

    • C++ Device API v2
      • The name function in the BankInterface, RegisterInterface and FieldInterface classes returns std::string_view now.
      • Added a third parameter offset for init method in Field class. The parameter should use the offset of the field on the containing register. This is an incompatible change, all invocations of the method need to be updated.
      • Added the WriteOnceRegister class for modeling a register which can only be written once.
      • Added the ReadOnlyClearOnReadRegister class for modeling a register which is read-only and the value is cleared on read.
      • Added the WriteOnceField class for modeling a field which can only be written once.
      • Added the ReadOnlyClearOnReadField class for modeling a field which is read-only and the value is cleared on read.
    • Core
      • The GDB distributed with Simics has been moved from Simics Base to a separate package, Simics-GDB with package number 1031. So please install the Simics-GDB package before starting bin/gdb on Linux or bin\gdb.bat on Windows.
    • Debugger
      • Change context property OperatingMode to OperatingModes (bug #18027361607).
      • Added SEAM mode to the OperatingModes context property of x86_64 cores (bug #18027361607).
    • Graphics console
      • The vnc_server interface is now deprecated. Use the vnc_server_v2 interface instead, and, if required, the tcp_connection_info interface on the tcp sub-object.
    • PCI DML library
      • Importing pci/common.dml no longer causes the device to implement io_memory. PCI devices still implement io_memory, as a side-effect of using the pci_common template.
    • PCIe DML library
      • Fixed typo on bitrange in dmwls field
    • PCIe Downstream Port
      • Fixed a bug in pcie-downstream-port-legacy where devices implementing pci_multi_function_device interface were not mapped and unmapped properly.
      • Devices implementing pci_multi_function_device interface are now considered 'transparent' by the pcie-downstream-port-legacy, if they are listed in the pci_devices attribute with device and function equal to zero.
    • Target parameters
      • Fixed a problem in argument override handling, which could result in some nested argument values being lost.
      • The list-targets and list-presets commands, the bin/list-targets script, and the simics -l output now displays the output in alphabetical order.
      • Fixed initialization order so that sim_params can be imported from simics_start.py (bug #14019229098).
      • Added target argument to the params.help command.
    • Text console
      • The telnet_connection interface is now deprecated. Use the telnet_connection_v2 interface instead, and, if required, the tcp_connection_info interface on the tcp sub-object.

  • 6.0.163 (build 6215)

    • C++ Device API v2
      • Added method parent on Register class which returns a pointer of BankInterface to its parent bank.
      • Added method parent on Field class which returns a pointer of RegisterInterface to its parent register.
      • Write to instance of ReadOnlyRegister class generates a spec_viol now instead of info log message.
      • Updated the default behavior of get and set methods in Register. Now get and set methods iterate over all fields' get and set methods in the ordering of ascending offset.
    • Core
      • Fixed a problem in embedded Simics, introduced in Simics-Base 6.0.160, where Python symbols were not available in the global namespace (bug #15013194653).
      • On Linux, Simics now only overrides LD_LIBRARY_PATH environment variable with SIMICS_LD_LIBRARY_PATH if the latter set (even if it is empty) (bug #15013235354).
      • The performance of the load-intel-obj command was improved (bug #14016107680).
      • The save-persistent-state command now shows saved state and file name when it returns (bug #18026558428).
      • The list-objects command no longer yields duplicate entries due to aliases, and will now display if any entry only exists due to a matching alias (bug #SIMICS-19195).
    • Device Modeling Language (DML) 1.2
      • Improve porting rules for method return values: In some simple cases, avoid storing the method's return value in an intermediate local variable.
    • PCI DML library
      • PCI devices now allow that the use_io_memory parameter of banks is set to false (bug #SIMICS-20988).
    • PCIe DML library
      • PCIe devices now allow that the use_io_memory parameter of msix_table banks is set to false (bug #SIMICS-20988).
      • Fixed typo, causing frss field to be added twice, when PCIe version is 6.
    • PCIe Downstream Port
      • Fixed a bug in pcie-downstream-port-legacy where devices implementing pci_multi_function_device interface were handled differently, compared to how the legacy pcie-bus handles them.
      • Fixed bug in info command when transparent devices are present.
    • Project Makefiles
      • Interface modules built with Simics-Base 6.0.160 or older will no longer work if they have method arguments or return values with types defined in Simics-Base, such as conf_object_t (excluding the mandatory first object arguments).
    • Python module: sloop
      • Fixed a crash in sloop.create_task. Also added a name parameter for consistency with asyncio counterparts.
    • Python
      • The Python library lxml has been removed from the Simics Base package, for security reasons (bug #SIMICS-20978).
    • Target parameters
      • Added an API function SIM_load_target that is similar to the load-target and run-script commands.
      • The load-target command now effectively uses the -local flag in the run-script command.
    • virtio-comp
      • The virtioFS PCIe component now has better documentation and is aligned with new feature in the virtioFS-fuse device.
    • virtiofs-fuse
      • The virtiofs fuse device now supports manual launch of the virtiofs daemon. The shared_dir has now changed name to share and has new added functionality. It can be set either to a directory which will be shared with the target, or a Unix domain socket file path of a virtiofs daemon instance.

Multi cell messaging will be scheduled as soon as possible at the destination cell, with messaging remaining asynchronous. A downside with immediate delivery for cross cell messaging is that it makes the simulation indeterministic as the cells run on different threads. 2 New Features of Simics 6

This section provides a high-level summary of all new features of Simics 6, presented per product.

Simics

Asynchronous Transactions
Support for non-blocking asynchronous memory transactions has been added. With asynchronous transactions, a target no longer has to respond to a memory access at once, i.e. in zero virtual time. Instead, the transaction can be completed later when data is available. This simplifies integrations with more detailed simulation models. Outstanding memory transactions can be inspected by the user and checkpointing of them is supported.
Instrumentation Framework
A new instrumentation infrastructure has been added, allowing flexible extraction of various run-time information from the system in a more efficient way than earlier. The instrumentation infrastructure supports CPU, DML and SystemC instrumentation. It provides generic commands for adding and controlling instrumentation in the simulation. Several example tools and filters are provided with source code as examples.
Device Register Coverage
Simics can generate information about register coverage, i.e. which registers in a system have been read/written during a session. This can for example be used to verify that expected parts of a system has been covered by tests.
Enhanced Target Consoles
There are new target consoles for both graphical and text (serial) output. The consoles clearly separate their backend (scripting, commands, breakpoints) from the presentation frontend. For graphics, a new VNC frontend as been added and there is improved support for rotating or scaling the display. The feature set of the new consoles is consistent over all supported host platforms.
3rd Generation Multi-Threaded Scheduler
The simulation thread scheduler has been greatly improved. A new thread domain concept has been introduced, improving performance of multi-threaded simulations through efficient cross-cell calls and simplifying the interaction with the Simics scheduler from user threads. The Multi-Core Accelerator functionality will be updated to take advantage of the new scheduler in an update to Simics 6, further improving performance of multi-threaded simulation workloads.
Hierarchical Objects
The configuration system has been extended with support for object hierarchies. Previously the component system had to be used to impose an hierarchical structure for objects. The hierarchical objects support is used to implement the new port objects and to simplify configuration of systems. Several CLI commands have been generalized to handle the object namespaces.
Notification System
A new light-weight notification system has been introduced that will eventually replace the current hap infrastructure.
Updated Host Requirements
The Windows version of Simics now requires Windows 8.1.
Deprecations and End-of-life
API and script compatibility with Simics 4.6 and older has been dropped. Refer to the Simics 6 Migration Guide for a complete list.

Simics Analyzer

Debugger API
An API for the source level debugger has been introduced, providing access to symbol and source information. The debugger API can for example be used with the new instrumentation framework to correlate events in the system with the target source code.

Simics Model Builder

DML 1.4
A new revision of Device Modeling Language (DML) is planned as a later addition to the Simics Model Builder product in Simics 6.
SystemC
The scheduler driving simulated time forward in SystemC has been updated, simplifying configuration as no external clock is needed. It also improves the interaction with standard commands in Simics related to simulated time.

Gaskets for standard protocols with default template parameters are provided as Simics classes, to support dynamic instantiation of gaskets based on meta data. There is also an interface for external tools to provide this meta data based on the name of the object. Both these techniques can be leveraged to support a GUI-based drag-n-drop configuration of generated SystemC modules and have them automatically wrapped by Simics.

Improved isolation between SystemC modules, so that SystemC models can be built using different compiler/tool chains and still be loaded into the same Simics session. This affects SystemC tools, which are now built and connected on a per module basis to avoid mistakes with the configuration.

The SystemC tools have been re-worked for improved consistency with other Simics tools.

Refer to the SystemC Library Programming Guide for information.

Global Time Queue
A new global time queue has been added to each cell in a simulation. This queue should be used instead of an arbitrary clock in the system when a global time queue is needed.
Port Objects
A new port object concept has been introduced that will eventually replace the existing port interfaces. Using the new hierarchical object support, one sub object, called port object, is created for each interface that an object implements. This removes the previous inconsistency in the configuration system where some implemented interfaces were named (i.e. port interfaces) and some were anonymous.

3 Detailed List of Changes Since 5

This section provides a list of minor changes since the Simics 5 version that was current at the time of the initial Simics 6 release.

Minor Changes

  • 6.0.0 (build 6003)

    • Classic GUI
      • Changed to Simics 6 icons.
    • Core
      • The SIM_copyright now returns the copyright string instead of printing it.
      • The -readme command line flag has been removed.
      • The attribute type Sim_Attr_Session is now deprecated. Use Sim_Attr_Pseudo instead.
      • The attribute type Sim_Attr_Doc is now deprecated. Attributes are always fully documented unless they are marked with Sim_Attr_Internal.
      • New simplified attribute registration functions SIM_register_attribute and SIM_register_class_attribute have been introduced. They do not support indexing or callback data, do not throw exceptions, require attribute type, do not support the a type in required attributes and do not support the D type. There are also functions SIM_register_attribute_with_user_data and SIM_register_class_attribute_with_user_data if callback data is needed. The old attribute registration functions SIM_register_typed_attribute and SIM_register_typed_class_attribute are now deprecated.
      • New list-interfaces command and wait-for-log command got the object argument. The namespace version of the commands was deprecated (bug #SIMICS-10496).
      • The mechanism to keep time synchronized between cells have been changed in order to decouple the round robin scheduling occurring within a cell from the cell synchronization mechanism used between cells. More specifically, only the first CPU in each cell is now subject to the requirement that time should be synchronized within a min latency quantum.
      • Restructured the documentation. Extension Builder User's Guide is now part of Model Builder User's Guide and Accelerator User's Guide is now part of the main Simics User's Guide. This reflects that Extension Builder and Accelerator are no longer separate Simics products.
      • list-attributes command will now return a list with attributes' names when used in the expression.
      • In Python, 'attr' namespace can be used to access Simics objects' attributes. The same namespace is also provided in pre_conf_object class (bug #SIMICS-10496).
      • The -py3k-warnings Simics command line parameter is now deprecated and has no longer any effect. Python warnings about incompatible code is instead printed by default. A parameter -no-py3k-warnings has been introduced that turns off warnings.
      • list-attributes, log, log-size, log-setup, log-type, (un)break-hap, (un)trace-hap commands got object parameter. The namespace version of the commands was deprecated (bug #SIMICS-10496).
      • The -stall and -fast flags to Simics have been deprecated. All functionality is available in the standard processor models, with no need to switch between the "stall" and "fast" versions.
      • A new transaction_t data type has been added to support asynchronous memory accesses. The old generic_transaction_t will remain fully supported in interfaces using it. It may be deprecated in a later major version of Simics.
      • The print-event-queue (peq) and print-time (ptime) commands now show the unscaled SystemC time and events, as well as the new picosecond queue.
      • It is possible to advance the simulation using the new picosecond queue (i.e. c 1 ps).
      • Instrumentation tools can be tied to classes and the new- command can be namespaced to that class.
      • Boost shared libraries (.so files) are no longer distributed with Simics.
      • Log output for write/read to/from registers in DML models are now zero padded to the full width of the registers (bug #SIMINT-898).
      • The run command now selects the picosecond queue when invoked with a time unit other than 'cycles' or 'steps'. The default behavior, when no time unit is supplied, is maintained for backwards compatibility.
    • Graphics console
      • Added commands show and hide, similar to the visible attribute.
    • OS Awareness Builder
      • The register_formatter has been changed so that it returns a cancel ID, which can be passed to the new unregister_formatter function (bug #SIMICS-11174).
    • Python Module: dev_util
      • The constructors of classes Register, Register_LE and Register_BE now accept object and offset as separate arguments, instead of a single argument with a tuple (object, bank, offset). The old form is still accepted for compatibility.
    • Standard Components
      • The console components are deprecated. Use the console-components module instead.
    • SystemC Library Documentation
      • Building with INTC_EXT=0 is no longer supported in Simics 6. Sections covering this has been removed from the documentation.
    • SystemC Library
      • Dynamic created tagged gaskets now use underscore instead of hyphes in their class name (bug #SIMICS-10284).
      • All gaskets provided by the SystemC Library now support re-entry. Gaskets written by the user must be updated manually.
      • On critical errors, the SystemC simulation will also be stopped preventing further execution of the SystemC scheduler.
      • Improved sanity check of Simics to SystemC interactions via custom adapter with custom gaskets.
      • Created the tool connection objects under the tool object namespace instead of in the global namespace (bug #SIMICS-11455).
      • In Simics 6, the SystemC configuration is wrapped by the same kind of Adapter meaning no need to update the source code. However, the Adapter now always creates a 'co-engine' object as a child of the Adapter. The co-engine works as a clock, and is scheduled directly by Simics. This means there is no longer any need for an external clock which will simplify both configuration of and interaction with the Adapter. The co-engine always runs via the picosecond queue, so no more scaled clocks and confusing log outputs. Standard CLI commands can be used directly on the adapter with improved clarity (i.e. print-event-queue and print-time).
      • Simics events can be posted directly on the adapter just like in 5, but it will be more transparent and with less glue code involved.
      • Because the co-engine is scheduled by Simics, the repostNextEvent and catchUp logic has been removed from the glue. All interactions takes place at the current time of the co-engine/SystemC scheduler. The SystemC scheduler may still need to run ahead of the co-engine (i.e. Simics) in order to satisfy the requirements of a synchronous Simics interface call.
      • The time_quantum attribute has been removed as it is not needed anymore with the improved scheduling.
      • Proxies are now created as hierarchical objects using the new hierarchical API. This avoids a few problems tied to the previous use of the component system to achieve similar results, and it also reduced the amount of glue code.
      • Static linking of Boost libraries to improve isolation of modules w.r.t. different C++ tool chains.
      • The instrumentation/tools have moved into SystemC Library and are built as part of the device module. This improves isolation of modules w.r.t. different C++ tool chains. The name of the tool classes are prefixed by the module's name to avoid confusion and invalid use when working with multiple SystemC integrations in the same Simics configuration.
      • The default instrumentation tool created when enabling or disabling tracing/breaking from a SystemC object has moved into the adapter's internal namespace. The reason for this is to achieve better isolation of user-created instrumentation tools.
      • Added run_next_delta attribute, to run the next delta cycle (or all cycles on current time)
    • SystemC Tools
      • The instrumentation/tools have moved into SystemC Library and are built as part of the device module. The tools are instantiated using namespaced commands of the adapter, not as global commands, to avoid confusion with other Adapters providing similar tools in the same configuration.
      • Hierarchical objects have been used ot hide the default tool created automatically by the namespaced trace-sc commands.
      • Added support for instrumentation filters and provided a standard filter supporting the following kinds of objects: events, process, signal, socket. See the instrumentation framework for more details.
      • The API has been improved to make the tools easier to use from CLI.
    • Text console
      • Added commands show and hide, similar to the visible attribute.
